# Face for Android

WARNING

Minimum Android Version: Lollipop 5.x (API level 21)

# Unzip the library

Face SDK for Android is an AAR library that contains two files:

  • libwebrtc-20211125.aar - Google WebRTC library
  • FACE.RELEASE-1.3.4-20211216.aar - The Face SDK itself

The first step is to unzip the contents into a folder of your choosing.

# Copy AAR files into libs folder

Your project's main app module should have a libs folder - that's the best place to copy the AAR files.

Open your app module's build.gradle file and include the following lines inside the dependencies section.

If the file doesn't have a dependencies section, check that you're in your app module's build.gradle file rather than that of your entire project.

dependencies {
    // aar files ...
    implementation files('libs/libwebrtc-20211125.aar')
    implementation files('libs/FACE.RELEASE-1.3.4-20211216.aar')

    // also need Websocket and JSON library 
    implementation 'org.java-websocket:Java-WebSocket:1.5.2'
    implementation 'com.alibaba:fastjson:1.1.73.android'
    // ...... something else
    implementation 'androidx.constraintlayout:constraintlayout:2.1.2'
}

# Permissions

The Face SDK requires the following permissions which you do not need to add as they add as they are automatically merged by including the SDK in the build.gradle dependencies:

<uses-permission android:name="android.permission.INTERNET"/>
<uses-permission android:name="android.permission.ACCESS_NETWORK_STATE"/>
<uses-permission android:name="android.permission.MODIFY_AUDIO_SETTINGS"/>

# Initialize SDK

The RTCPlayerClient.initialize() that used to initialize environment of WebRTC should be called as soon as possible and only once for the entire life of the activity.

So it's recommended to call this method in Application::onCreate().

package ai.zhuiyi.demo

import ai.zhuiyi.face.webrtc.RTCPlayerClient
import android.app.Application

class FaceApp : Application() {
    companion object;

    override fun onCreate() {
        super.onCreate()

        RTCPlayerClient.initialize(applicationContext)
    }
}

# Declaring a layout

Create a new XML layout of player view in res/layouts/player.xml using ConstraintLayout. Example:

<?xml version="1.0" encoding="utf-8"?>
<androidx.constraintlayout.widget.ConstraintLayout
    xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:app="http://schemas.android.com/apk/res-auto"
    xmlns:tools="http://schemas.android.com/tools"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    tools:context="ai.zhuiyi.demo.PlayerActivity"> <!-- change package name if needed-->

    <org.webrtc.SurfaceViewRenderer
        android:id="@+id/webrtc_view"
        android:layout_width="match_parent"
        android:layout_height="match_parent"
        android:layout_gravity="center"
        android:visibility="gone" />
</androidx.constraintlayout.widget.ConstraintLayout>

Then create a PlayerActivity and initialize the SDK in Activity::onCreate().

# Initialize Player

private lateinit var view: SurfaceViewRenderer
private lateinit var playerClient: RTCPlayerClient

view = findViewById<SurfaceViewRenderer>(R.id.webrtc_view)

playerClient = RTCPlayerClient.Builder()
    .fromEndpoint(endpoint)
    .setHttpApiPort(port)
    .setView(view)
    .create(applicationContext, EglBase.create())

The parameters of fromEndpoint() and setHttpApiPort() for creating an instance of RTCPlayerClient will be provided by the factory's technicians.

The Face SDK supports the Android Lolipop or higher. But it is better to check the version of the system before using it.

if (!RTCPlayerClient.isSupported()) {
    Log.e(TAG, "Your device is not supported");
    return;
}

    }

    override fun onPlayStart(text: String?) {

    // Called when the digital human stop talking
    override fun onPlayEnd(text: String?) {
    }

    // Called when receiving a websocket message (json) from the control server
    override fun onMessage(msg: String?) {
    }

    // Called when the control server is disconnected
    override fun onSocketClosed(code: Int, reason: String?) {
    }

    // Called when the control server is connected incorrectly or the network is disconnected
    override fun onSocketError(ex: Exception?) {
    }

    // Called fps detection
    override fun onStatsUpdated(fps: int) {
    }
})

All the available callback functions are listed above. Be careful that some callback functions should be called in non-UI threads, such as onFirstFrameRendered,

So if you want to update the interface, you need to dispatch the code to the main thread (for example, use: runOnUiThread {}) to avoid ANR problems.

All callback functions do not need to call super.xxxx().

# Lifecycles

Finally, the same method of RTCPlayerClient should be called in the life cycle method of PlayerActivity.

override fun onStart() {
    super.onStart()
    playerClient.start()
}

override fun onStop() {
    super.onStop()
    playerClient.stop()
}

override fun onResume() {
    super.onResume()
    playerClient.resume()
}

override fun onDestroy() {
    super.onDestroy()
    playerClient.release()
}

That's all.
