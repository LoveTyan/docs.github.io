# 追一数字人IOS端集成流程

WARNING

SDK 目前支持的版本为 iOS 10+

# 安装 SDK

在苹果系统中，SDK 将会以 framework 的形式提供，通常会包含：

  • WebRTC.framework
  • Vibot.xcframework

# 添加依赖

打开您的项目工程，进入项目名进入设置界面，依次选择：

    TARGETS -> (App) -> General

找到 Frameworks, Libraries and Embedded Content 部分，将提供的两个包拖进去即可。

或者直接拖到左侧导航栏的 Frameworks 目录下，在弹出的界面中选择要添加到的 Target 即可。

# 权限声明

不需要声明权限。

# 界面渲染

首先我们需要一个可渲染动画的控件，Google WebRTC 为我们提供了两个选择：

  • 基于 OpenGL ES 的 RTCEAGLVideoView
  • 基于 Metal 的 RTCMTLVideoView

如果支持的话，SDK 内部会自动选择 RTCMTLVideoView，否则会自动降到 RTCEAGLVideoView，不需要用户自动检测。

只要在您的视图类中定义属性时，使用 RTCRenderView 替代即可：

@property(nonatomic, strong) RTCRenderView *myview;

为了防止报错，需要将两个头文件都导入：

#import <WebRTC/RTCEAGLVideoView.h>
#import <WebRTC/RTCMTLVideoView.h>

初始化视图对象，赋值给 RTCPlayer 的 setView：

_myview = [[RTCRenderView alloc] initWithFrame:CGRectZero];
[self.view addSubview:_myview];

# 初始化

与安卓系统不同，iOS 系统中初始化过程比较简单，比如在 viewDidLoad 方法中运行如下代码：

// 检查iOS系统是否支持数字人
if (![RTCPlayer isSupported]) {
    NSLog(@"Not supported");
    return;
}

// 初始化配置信息
RTCPlayerConfig *config = [RTCPlayerConfig shared];
[config setHost:@"example.wezhuiyi.com"];
[config setStreamHost:@"example.wezhuiyi.com"];
[config useSecureProtocol:YES];

// 初始化播放器
_player = RTCPlayer.shared;
[_player setConfig:config];
[_player setDelegate:self];
[_player setView:_myview];
[_player setBounds:self.view.bounds];
[_player start];

isSupported 用来检查当前运行的系统是否支持数字人，比如系统版本小于 iOS 10 的将无法使用数字人， 某些旧型号的苹果手机，比如 iPhone5 和 iPhone5C 也会被屏蔽。

RTCPlayerConfig 用来设置配置信息。

setView 传入渲染视图的实例。

setBounds 传入视频显示区域的边界大小，通常是渲染控件的父容器的大小。

# 委托方法

为了有效的控制视频以及实现相关业务逻辑，SDK 定义了很多回调方法，在定义视图控制器时需要实现这些委托方法：

@interface PlayerViewController ()<RTCPlayerDelegate>

以下是 RTCPlayer 的所有委托方法：

// 委托
@protocol RTCPlayerDelegate

// 流媒体服务器断开时触发
// 网络连接断开时触发
// 立即响应
- (void)onConnectionDisconnected;

// 超过一定时间没有任何动作，自动关闭播放器
- (void)onIdleTimeout;

// 网络状态
- (void)onNetworkStatus:(RTCNetworkStatus)status;

// Websocket 连通后，接收到中控发来的播放地址
- (void)onReceivePlayURL:(NSString *)url;

// 接收到视频服务器发送来的视频信号
- (void)onReceiveStream:(RTCVideoTrack *)track;

// 语音播报开始
- (void)onPlaySoundStart:(NSString *)msg;

// 语音播报结束
- (void)onPlaySoundEnd:(NSString *)msg;

// 收到来自中控服务器的消息
- (void)onMessage:(NSString *)msg;

// 中控服务器断开时触发
// 网络断开时触发
// 有延迟
- (void)onSocketClosed:(NSInteger)code reason:(NSString *)reason wasClean:(BOOL)wasClean;;

@end

部分方法在非 UI 线程中调用，所以如果要更新界面，需要切换到主线程。如：

dispatch_async(dispatch_get_main_queue(), ^{
    // your code
});

# 屏幕适配

数字人的原始分辨率为 540 * 960，由于手机设备的高宽比各不相同，所以在非 16:9 的设备上可能会出现白边的情况。 SDK 在内部已经自动调整画面按照比例缩放填充整个播放区域。

# 资源释放

只需调用 stop 方法即可。

- (void)viewWillDisappear:(BOOL)animated {
    [super viewWillDisappear:animated];
    [_player stop];
}
