# 追一数字人 HTML5集成流程

TIP

SDK 支持目前绝大部分的主流浏览器，比如 Chrome, Edge, Safari 等...也可在微信小程序中使用。

# 安装 SDK

SDK 会以压缩包的形式提供，类似如下的格式：

  • face-html-2022xxxx.zip

# 添加依赖

本项目暂时未发布到公网的 npm 仓库，所以使用本地安装的方式添加依赖。

将 SDK 压缩包解压缩到项目目录，然后使用 npm install 命令链接：

npm install face-html/

这个命令会在 node_modules 目录中创建一个软连接指向到实际的目录。当然也可以直接复制 face-html 目录到 node_modules 中，但是不推荐这么做。 因为 npm 的命令在更新依赖时会清理所有未在 package.json 中注册的模块文件。

您也可以使用 npm link 命令安装依赖。

另外由于使用了 @babel/plugin-transform-runtime 辅助开发，所以需要添加此插件的运行时依赖：

npm install --save @babel/runtime

检查项目 package.json 中的 dependencies 部分：

{
    "devDependencies": {
        //
    },
    "dependencies": {
        "@babel/runtime": "^7.0.0",
        "face-html": "file:./face-html"
        //...
    }
}

# 示例代码

以下会以一个基础的 Vue 页面为例，讲解接入方式。

您可以使用 vue serve hello.vue 命令快速运行一个 Vue 页面，无需创建一个独立的项目。 具体方式请参考：https://cli.vuejs.org/guide/prototyping.html

# 播放器

首先在页面中嵌入一个播放器：

<template>
    <video id="rtc_player" autoplay muted playsinline />
</template>

# 连接数字人

导入 FaceClient，实例化以后设置属性 endpoint 的值，也就是中控服务器的 Websocket 连接地址，调用 start() 方法启动即可。

然后我们只要关注对象的各个回调方法，即可与服务器进行交互。

import { FaceClient } from 'face-html'

export default {
    data() {
        return {
            player: null,
        }
    },

    mounted() {
        // ...

        // 初始化播放器并连接中控
        this.player = new RTCPlayerExpress()
        this.player.secure = false
        this.player.endpoint = `ws://....`
        this.player.start()

        // 关联视频流
        this.player.onTrack = event => {
            // 通常会接收到来自服务器的有两条流 video 和 audio
            // 只需要关联 video 即可
            if (event.track.kind != 'video') {
                return
            }

            var media = document.getElementById('rtc_player')
            if (media && event.streams.length > 0) {
                media.srcObject = event.streams[0]
            }
        }

        // 监听来自服务器的交互信息
        this.player.onMessage = message => {
            console.log(message)
        }

        //  启动 5 秒后，发送一条信息
        setTimeout(() => {
            const msg = JSON.stringify({ type: 5, video_text: '您好，我是追一数字人...' })
            this.player.send(msg)
        }, 5000)
    }

# 公共函数以及回调方法

以下是 TypeScript 定义的目前可用的接口以及一些回调方法，使用方式可参考上面的示例。

    // 启动数字人
    start(): void
    // 关闭数字人
    close(): void
    // 发送文本
    send(message: string): void

    // 当收到视频流时触发
    onTrack!: (event: RTCTrackEvent) => void
    // 当中控连接超时时触发
    onConnectionDisconnected!: () => void
    // 空闲时间超过 600 秒，退出系统并且触发此方法
    onIdleTimeout!: () => void
    // 数字人开始语音播报时
    onPlayStart!: (message: string) => void
    // 数字人结束语音播报时
    onPlayEnd!: (message: string) => void
    // 当收到来自服务器的消息时触发
    onMessage!: (message: object) => void
    // 中控服务器断开时
    onSocketClosed!: (code:Int, reason: string) => void
    // 中控服务器连接错误时
    onSocketError!: (error: string) => void

# 浏览器兼容性

由于客户端上设备的多样性，前端环境较为复杂，即使相同的代码在不同的环境中也可能表现出差异性。

如 video 标签，在不同浏览器中行为受到不同的限制，Google Chrome 会严格限制音视频的自动播放，需要用户的交互行为或是使用 muted 标签关闭声音。

又或者在苹果 iOS 系统中，系统总是会尝试以全屏的方式播放视频，如果失败则自动暂停，添加 playsinline 标签可使视频可以在未全屏的情况下自动播放。

诸如此类，与安卓等平台不同时，H5 SDK 无法干预客户端播放器的行为，此类问题需要前端的开发者自行处理。
